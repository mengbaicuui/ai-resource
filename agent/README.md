## 1. Claude Code

![Anthropic](https://www.anthropic.com)

### 基本信息

| 项目 | 内容 |
|------|------|
| **开发公司** | Anthropic |
| **官网** | claude.ai/claude-code |
| **定位** | CLI 编程助手 |
| **发布时间** | 2024年 |

### 底层技术

#### 🔧 核心技术栈
- **底层模型**: Claude 3.5 Sonnet / Opus
- **核心协议**: MCP (Model Context Protocol) - Anthropic 主导开发的开放协议
- **工具调用**: 自主研发的 Tool Use 框架，支持搜索、文件操作、bash 执行
- **架构模式**: 单 agent 循环执行模式

#### 🧠 技术特点

```
用户指令 → Claude (Haiku/Sonnet/Opus) → Tool Execution → 循环直到完成
                ↓
         MCP Server (扩展工具生态)
```

1. **MCP (Model Context Protocol)**
   - 开放协议，定义 LLM 如何与外部工具交互
   - 支持 Server-Client 架构，可扩展性强
   - 类似 JSON-RPC，但专为 AI 场景优化

2. **Tool Use 框架**
   - 内置工具：Search、Bash、Read、Write、Edit、Notebook
   - 权限分级：每次操作需用户确认
   - 支持沙箱执行环境

3. **文件操作**
   - 支持 glob 模式匹配
   - 增量编辑（diff-based）
   - 多文件协同修改

### 💡 优势

- 强大的代码理解和生成能力
- 开放的 MCP 生态，方便扩展
- 支持多种 Claude 模型（Haiku 高效 / Sonnet 平衡 / Opus 最强）
- 完善的测试和构建流程自动化

### ⚠️ 局限

- 主要面向 CLI，IDE 集成较弱
- 需要手动确认每个敏感操作
- 上下文窗口受模型限制

## 2. Cursor

![Cursor](https://cursor.com)

### 基本信息

| 项目 | 内容 |
|------|------|
| **开发公司** | Cursor (Anysphere) |
| **官网** | cursor.com |
| **融资** | Y Combinator 孵化，估值超10亿美元 |
| **用户量** | 超50万开发者，财富500强广泛采用 |

### 底层技术

#### 🔧 核心技术栈

| 层级 | 技术 |
|------|------|
| **底层模型** | OpenAI GPT-4o、Anthropic Claude 3.5、xAI Grok、Google Gemini、Cursor 自研 |
| **架构模式** | Multi-Agent 多智能体系统 |
| **上下文** | Whole Codebase Understanding (代码库全量理解) |
| **扩展协议** | MCP 支持 |

#### 🧠 技术特点

```
┌─────────────────────────────────────────────┐
│              Cursor IDE                       │
├─────────────────────────────────────────────┤
│  Tab Completion    Cmd+K     Agent Mode       │
│      ↓              ↓           ↓          │
│  单行补全      定向编辑    自主执行任务       │
├─────────────────────────────────────────────┤
│           Multi-Agent Orchestrator           │
│    (协调多个 Agent 并行/串行工作)            │
├─────────────────────────────────────────────┤
│  Model Router (智能路由，选择最佳模型)       │
├─────────────────────────────────────────────┤
│  GPT-4o │ Claude 3.5 │ Gemini │ Grok │ 自研  │
└─────────────────────────────────────────────┘
```

1. **Multi-Agent 架构**
   - 多个专业 Agent 并行工作
   - 代码编写、测试、调试、重构各有分工
   - Agent 间可通信协作

2. **Codebase Intelligence**
   - 支持任意规模的代码库理解
   - 语义索引，快速定位相关代码
   - 学习项目模式和编码风格

3. **Autonomy Slider (自主度滑块)**
   - Tab: 仅补全
   - Cmd+K: 定向编辑
   - Agent: 完全自主执行

### 💡 优势

- 首个将多 Agent 集成到 IDE 的产品
- 支持几乎所有顶级 LLM
- 企业级安全，Fortune 500 信任
- 预览模式：自动启动服务器，实时预览效果

### ⚠️ 局限

- 付费产品（Pro 订阅）
- 部分功能需要 Copilot++ 订阅

## 3. Windsurf

![Windsurf](https://windsurf.com)

### 基本信息

| 项目 | 内容 |
|------|------|
| **开发公司** | Codeium |
| **官网** | windsurf.com |
| **产品名** | Cascade (AI 引擎) |
| **用户量** | 100万+ 用户，4000+ 企业客户 |

### 底层技术

#### 🔧 核心技术栈

| 层级 | 技术 |
|------|------|
| **底层模型** | 支持主流 LLM（具体提供商未公开） |
| **特色功能** | Memories、RULE、Cascade |
| **扩展协议** | MCP (Model Context Protocol) |
| **架构模式** | 单 Agent + 记忆系统 |

#### 🧠 技术特点

1. **Cascade AI 引擎**
   ```
   用户指令 → Cascade (记忆上下文) → 执行 → 持续学习
                   ↓
            Memories (项目记忆)
            Rules (自定义规则)
   ```

2. **Memories (记忆系统)**
   - 跨会话记住项目结构和偏好
   - 理解开发者的工作流习惯
   - 持续积累项目上下文

3. **Rules (规则引擎)**
   - 开发者可定义编码规范
   - AI 严格遵循项目规范
   - 支持框架特定规则（如 Next.js 模式）

4. **MCP 支持**
   - 一键连接 MCP Servers
   - 内置 Figma、Slack、Stripe、GitHub、Postgres 等集成

5. **Turbo Mode**
   - AI 自主执行终端命令
   - 自动检测并修复 lint 错误
   - 自动化工作流

### 💡 优势

- **免费使用**：对个人开发者友好
- **智能记忆**：越用越懂你的项目
- **JetBrains 插件**：首个官方 AI 插件入驻 JetBrains IDE
- **实时预览**：内置服务器，拖拽图片即可生成 UI

### ⚠️ 局限

- 企业功能相对较弱
- 模型选择不如 Cursor 灵活

## 9. 技术趋势总结

### 🔮 核心技术趋势

#### 1. Multi-Agent 系统成为主流

```
单一 Agent                    多 Agent 协作
┌─────────┐                  ┌─────────┐
│  规划   │                  │ 规划 Agent │
└────┬────┘                  ├─────────┤
     │                        │编码 Agent │
┌────┴────┐                  ├─────────┤
│  执行   │                  │测试 Agent │
└─────────┘                  ├─────────┤
                              │审查 Agent │
                              └─────────┘
```

**代表产品**: Cursor、GitHub Copilot

#### 2. MCP 协议统一工具生态

- ** Anthropic 主导的开放协议**
- 类似于 USB 接口之于硬件
- 一套标准，多种工具即插即用
- 生态快速扩张中

#### 3. 自主度连续光谱

| 自主度 | 模式 | 适用场景 |
|--------|------|----------|
| 0% | Tab 补全 | 新手、精确控制 |
| 30% | Cmd+K 定向编辑 | 日常开发 |
| 70% | 半自动 Agent | 复杂任务 |
| 100% | 全自主 Agent | 端到端项目 |

#### 4. 上下文能力持续增强

- **代码库全量理解**：从单文件到整个项目
- **跨会话记忆**：记住项目偏好和上下文
- **长期任务维持**：数小时复杂任务不中断

#### 5. 安全与权限分级

- 操作权限分级确认
- 敏感操作需人工授权
- 企业级数据保护
