## 1. 底层核心技术

### A. Spec-Driven Development (规范驱动开发)

Kiro 的核心差异化：**将自然语言需求转化为结构化规范，再由 Agent 执行**。

```
┌─────────────────────────────────────────────────────────────┐
│                Spec-Driven Development 流程                    │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   用户自然语言需求                                           │
│          ↓                                                  │
│   Kiro 生成结构化规范 (Spec)                                │
│          ↓                                                  │
│   任务分解 → 设计 → Agent 执行                              │
│          ↓                                                  │
│   结果验证 → 人工确认                                       │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### B. Agent Hooks (代理钩子)

| 特性 | 说明 |
|------|------|
| **事件触发** | 文件保存等事件自动触发 |
| **后台执行** | Agent 自主在后台运行 |
| **自动化任务** | 文档生成、单元测试、性能优化 |

---

## 2. 核心机制

### A. Steering (导航系统)

| 特性 | 说明 |
|------|------|
| **Steering Files** | 定义 Agent 与项目的交互方式 |
| **上下文注入** | 添加编码标准、工作流偏好 |
| **项目规则** | 简单命令配置Agent行为 |

### B. Autopilot Mode (自动驾驶)

| 特性 | 说明 |
|------|------|
| **自主执行** | Agent 自主完成大型任务 |
| **无需步骤** | 不需要逐步指令 |
| **保持控制** | 人类保留最终审批权 |

### C. 模型选择

| 模式 | 说明 |
|------|------|
| **Claude Sonnet 4.5** | 可靠的高级编码和推理 |
| **Auto** | 智能混合多模型，优化质量/延迟/成本 |

## 3. 与其他主流 Agent 工具的异同

### 横向对比

| 维度 | Claude Code | Cursor | Kiro | OpenCode |
|:---|:---|:---|:---|:---|
| **核心范式** | Agentic Loop | Autonomy Slider | **Spec-Driven** | 多模型 |
| **交互触发** | 命令驱动 | 模式切换 | **事件触发** | 命令驱动 |
| **AWS 背书** | ❌ | ❌ | **✅** | ❌ |
| **Agent Hooks** | ❌ | ❌ | **✅** | ❌ |
| **Steering** | ⚠️ | ⚠️ | **✅** | ❌ |
| **免费额度** | ✅ | ❌ | 50 credits | ✅ |

---

### 关键异同点

#### Kiro vs. Claude Code

| 同 | 异 |
|---|---|
| 都强调 Agent 能力 | Claude Code 是 Agentic Loop；Kiro 是 **Spec-Driven**，更结构化 |

#### Kiro vs. OpenCode

| 同 | 异 |
|---|---|
| 都是开源 CLI | Kiro **AWS 背书**；Kiro 有 Agent Hooks 和 Steering |

## 5. PM 视角总结

### 核心价值

| 价值点 | 说明 |
|--------|------|
| **Spec-Driven** | 结构化需求转化，避免 AI 误解需求 |
| **AWS 背书** | Amazon 官方，技术可靠 |
| **Agent Hooks** | 创新性的事件驱动自动化 |
| **模型优化** | Claude Sonnet 4.5 + Auto 路由 |

### 适用场景

- ✅ 需要结构化开发流程
- ✅ 大型项目需要规范约束
- ✅ 喜欢事件驱动自动化
- ✅ AWS 生态用户

### 不适用场景

- ⚠️ 喜欢自由发挥的快速原型
- ⚠️ 需要最强推理（Opus）
- ⚠️ 预算敏感（按 credits 计费）
